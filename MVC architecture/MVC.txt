what is MVC 
MVC stands for : M=model , V=view , C=controller
It is a way to organize the code so that project is clean, scalable, easy to debug, easy to maintain

why MVC needed ?
to avoid bad structure we need MVC

MVC components ?
1. Model
Handles data and logic
Interacts with database
Example: storing user details

👉 Think: “Data part”

🔷 2. View
Handles UI (User Interface)
What the user sees (HTML, UI pages)

👉 Think: “Display part”

🔷 3. Controller
Acts as a middleman
Takes user input → processes it → updates Model & View

👉 Think: “Control part”


MVC Flow (Model–View–Controller)

The MVC flow shows how data moves inside an application 💻

📊 Step-by-step flow:
1.User interacts with View
Example: clicks a button or submits a form
2.View sends request to Controller
Controller receives user input
3.Controller processes the request
Decides what to do (logic)
4.Controller updates/requests data from Model
Model handles database or data logic
5.Model sends data back to Controller
6.Controller sends data to View
7.View displays the result to the User

Simple Flow Line:

User → View → Controller → Model → Controller → View → User

Real Example (Login):

User enters login → View
Controller checks input
Model verifies data (DB)
Controller gets result
View shows “Login Success / Error”

Used in frameworks:

Express.js
Django
Laravel


-----Error Handling----------

Handles errors without crashing the app
Uses status codes (404, 500, etc.)
Use try...catch in controllers
Use one common error middleware
Can create custom errors (like “Student not found”)

👉 Purpose: Show proper error message to user

--------Logging------------
Records request details (GET, POST, URL)
Helps in debugging and tracking
Done using middleware

👉 Example log:
GET /api/students

🔄 Simple Flow

User → Controller → Error → Middleware → Response

🧠 One-line answer:
Error Handling = manage errors safely
Logging = track what happens in app
